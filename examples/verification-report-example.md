## Critical Issues (must fix before any sharing)

### 1. Sign error in Eq. 5.4 (Lens 1: Math)

**Location**: Section 5.4, page 18, Equation (5.4)

**Issue**: The interaction term in the DiD specification is written as:

```
Y_ist = alpha + beta_1 * NJ_s + beta_2 * Post_t - beta_3 * (NJ_s x Post_t) + X_ist'gamma + epsilon_ist
```

The minus sign on beta_3 is inconsistent with the interpretation in Section 5.5, which states: "The coefficient beta_3 = 2.76 represents the *increase* in FTE employment attributable to the minimum wage." If beta_3 enters with a negative sign in the equation, then a positive beta_3 estimate implies the treatment *reduced* employment -- the opposite of what the text claims.

**Fix**: Change the minus sign to a plus sign in Eq. 5.4, or redefine beta_3 with the opposite sign and adjust the interpretation. The standard DiD specification uses a positive sign on the interaction term: `+ beta_3 * (NJ_s x Post_t)`.

**Propagation risk**: Check that all subsequent references to beta_3 in Sections 5.5-5.7 are consistent with the corrected sign convention. The simulation code in `ch05_did.py` uses a positive sign (line 87: `treat * post`), so the code is correct -- only the equation in the text is wrong.

### 2. Sample size mismatch between text and code output (Lens 2: Empirical claims)

**Location**: Section 5.3, paragraph 1 vs. Table 5.1

**Issue**: The text states: "Our replication sample includes **410 fast-food restaurants**, matching the original Card and Krueger (1994) survey." However, Table 5.1 (generated by `ch05_did.py`) reports N = 399 in all columns. The code drops 11 observations due to missing FTE data in wave 2 (line 52: `df.dropna(subset=['fte_wave2'])`). This cleaning step was added on March 18 after the text was last edited on March 12.

**Fix**: Either (a) update the text to read "399 restaurants with complete data in both waves (of 410 surveyed)" or (b) revise the code to impute missing wave 2 FTE values rather than dropping observations, then regenerate Table 5.1. Option (a) is simpler and honest. Also add a sentence noting the 11 dropped observations and their characteristics.

## Important Issues (fix before submission)

### 3. Parallel trends not tested for teen subgroup (Lens 3: Causal logic)

**Location**: Section 5.6, "Heterogeneous Effects by Age Group"

**Issue**: Section 5.6 presents separate DiD estimates for teen workers (ages 16-19) and adult workers (ages 20+), finding a statistically significant positive employment effect for teens (beta = 4.12, SE = 1.89) and a null effect for adults. However, the parallel trends validation in Section 5.4 uses the full sample only. If pre-trends differ for teens vs. adults (plausible, since teen employment is more seasonal and volatile), the teen-specific estimate could be driven by differential trends rather than the minimum wage. A subgroup-specific pre-trend analysis is expected by modern DiD standards.

**Fix**: Add an event-study plot or pre-trend test for the teen and adult subgroups separately. If teen pre-trends are not parallel, acknowledge this limitation explicitly and downweight the subgroup finding.

### 6. Untested pre-treatment covariates balance (Lens 3: Causal logic)

**Location**: Section 5.3

**Issue**: The chapter describes the NJ and PA samples' pre-treatment characteristics (chain composition, wages, hours) but does not present a formal balance test (e.g., a normalized differences table or t-tests for pre-treatment means). While the samples appear similar from the descriptive statistics, a balance table is expected for a DiD chapter and provides a stronger foundation for the parallel trends assumption.

**Fix**: Add a balance table (Table 5.0 or Appendix Table A1) reporting pre-treatment means, differences, and normalized differences for key covariates across NJ and PA restaurants.

## Minor Issues (fix when convenient)

### 7. Notation inconsistency: subscript convention (Lens 1: Math)

**Location**: Eqs. 5.1 and 5.4

**Issue**: Eq. 5.1 uses `Y_it` (individual-time) but Eq. 5.4 uses `Y_ist` (individual-state-time). The state subscript `s` is redundant since each restaurant `i` belongs to one state, but if included, it should appear consistently throughout. The NOTATION.md file (if it exists) should specify the convention.

**Fix**: Use `Y_it` throughout (simpler) or `Y_ist` throughout (more explicit). Be consistent.

### 8. Rounding inconsistency in coefficient reporting (Lens 2: Empirical claims)

**Location**: Section 5.5, paragraphs 2-3

**Issue**: The main DiD coefficient is reported as "2.76" (two decimal places) in paragraph 2 but as "2.8" (one decimal place) in paragraph 3. Both are correct roundings of the point estimate (2.757), but the inconsistency looks careless.

**Fix**: Use two decimal places consistently throughout: "2.76" everywhere.

### 9. Figure 5.2 axis label truncated (Lens 2: Empirical claims)

**Location**: Figure 5.2, "Event Study Plot"

**Issue**: The y-axis label reads "FTE Emplo..." -- it appears the label string was truncated when the figure was exported at the current width. The axis title should read "FTE Employment (relative to t = -1)".

**Fix**: In `code/fig_event_study.py`, increase the figure width from 6 to 8 inches or abbreviate the label to "FTE Employment (rel. to t=-1)".

### 10. "Percent" vs. "percentage point" ambiguity (Lens 6: Overclaiming)

**Location**: Section 5.7, paragraph 2

**Issue**: The text states "employment increased by approximately 13 percent." The 13% figure comes from dividing the DiD estimate (2.76 FTE) by the PA control group's base employment (~21 FTE). This is a *percentage* change, not percentage points, and the phrasing is technically correct. However, in the same paragraph, "wages rose by 15 percent" is used to describe the minimum wage increase, which is also a percentage change. A reader could confuse these magnitudes. Consider adding explicit clarification.

**Fix**: Revise to "FTE employment increased by approximately 2.8 workers per restaurant, or roughly 13 percent of the control group mean" to eliminate ambiguity.

### 11. Unnecessary "ibid" usage (Lens 4: Citations)

**Location**: Footnote 14

**Issue**: Footnote 14 uses "ibid." to refer to Card and Krueger (1994). While not technically wrong, many economics style guides (including AER) discourage "ibid." in favor of repeating the author-date citation. This is a stylistic preference, not an error.

**Fix**: Replace "ibid." with "Card and Krueger (1994)" for clarity.

### 12. Inconsistent thousands separator (Lens 1: Math)

**Location**: Table 5.1, column headers vs. body

**Issue**: The table header says "N = 12,450" (with comma) for person-level observations in one specification, but the code output file uses "12450" (no comma). The rendered table should use consistent formatting.

**Fix**: Ensure the LaTeX table template applies `\num{}` formatting (from siunitx) or manually add commas to all numbers above 999.
