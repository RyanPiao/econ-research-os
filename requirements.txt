# /status — Project Dashboard

## Procedure

1. Identify the project root (look for `_quarto.yml`, `CLAUDE.md`, or `outline.md`)

2. Scan and report:

### Chapters
```bash
# Find all chapter files and check their YAML status field
for f in chapters/*.qmd; do
  grep -m1 "status:" "$f" 2>/dev/null
done
```
Report as table: Chapter | Title | Status (draft/reviewed/verified/compiled)

### Knowledge base
```bash
# Count paper snapshots
ls literature/*.md 2>/dev/null | wc -l

# Count by status
grep -rl "status: detailed" literature/ | wc -l   # read
grep -rl "status: queued" literature/ | wc -l      # queued
```

### Stale outputs
```bash
# Find code files newer than their output
for script in code/*.py; do
  output=$(grep -m1 "output/" "$script" | grep -oP '"[^"]*"' | tr -d '"')
  if [ -n "$output" ] && [ -f "$output" ]; then
    if [ "$script" -nt "$output" ]; then
      echo "STALE: $script → $output"
    fi
  fi
done
```

### Bibliography
```bash
# Count .bib entries
grep -c "^@" references.bib 2>/dev/null

# Find cited but missing keys
grep -ohP '@\w+' chapters/*.qmd | sort -u > /tmp/cited.txt
grep -ohP '^\@\w+\{(\w+),' references.bib | sed 's/.*{//;s/,//' | sort -u > /tmp/bibkeys.txt
comm -23 /tmp/cited.txt /tmp/bibkeys.txt  # missing
```

### Next actions
Infer from current state:
- If any chapter has status "draft" but not "reviewed" → "Run /self-review on Chapter [N]"
- If any chapter has status "reviewed" but not "verified" → "Run /verify on Chapter [N]"
- If stale outputs exist → "Re-run [script] to update [output]"
- If reading list has queued papers → "Read next paper: [title]"

3. Display as compact dashboard. Save to `./STATUS.md` (overwritten each time).

# /bib — Bibliography Management

## /bib add [DOI | URL | title]

1. Resolve the input:
   - If DOI: fetch metadata from Crossref or Semantic Scholar
   - If URL: extract DOI or title, then fetch metadata
   - If title: search Semantic Scholar, confirm match with user

2. Format as BibTeX entry:
```bibtex
@article{AuthorYear,
  author  = {Last, First and Last2, First2},
  title   = {Full Title},
  journal = {Journal Name},
  year    = {2024},
  volume  = {XX},
  number  = {X},
  pages   = {XX--XX},
  doi     = {10.XXXX/XXXXX}
}
```

3. Check for duplicates in references.bib (match on DOI or title similarity)
4. Append to references.bib if not duplicate
5. Confirm: "Added @AuthorYear to references.bib"

## /bib check [chapter-file]

1. Extract all `[@citekey]` from the chapter
2. Check each against references.bib
3. Report: missing keys, unused entries, incomplete entries (missing year/journal)

## /bib dedup

1. Parse all entries in references.bib
2. Group by DOI (if present) or fuzzy title match
3. Report duplicates with line numbers
4. Offer to merge (keep the more complete entry)

## /bib update-wp

1. For each entry with type `@techreport` or `@unpublished`:
   - Search Semantic Scholar for the title
   - Check if a published journal version now exists
2. Report: "Working paper @AuthorYear now published in [Journal] [Year]"
3. Offer to update the entry
